# moodle-mcp: Plug-and-Play Setup Wizard

**Date:** 2026-04-18
**Status:** Approved

---

## Goal

Make `npx moodle-mcp` work out-of-the-box for a complete stranger discovering the project on Reddit. Zero manual config editing. One command.

The package is published on npm as `moodle-mcp` (v0.1.0).

---

## What We're Building

Three deliverables:

1. **Setup wizard** — interactive first-run flow in the terminal
2. **`moodle_build_obsidian_vault` MCP tool** — on-demand vault generation from Claude
3. **README overhaul** — leads with one command, no friction

---

## 1. Setup Wizard

### Trigger condition

`server.ts` entry point checks on startup:
- No config exists (no env vars, no `~/.moodle-mcp.json`)
- `process.stdin.isTTY === true` (running in a real terminal, not launched by Claude Desktop)

Both must be true. If either is false, skip the wizard and start the MCP server normally (or throw the existing error if no config at all and no TTY).

### Config resolution order

1. Environment variables (`MOODLE_URL`, `MOODLE_TOKEN`, `MOODLE_USERNAME`, `MOODLE_PASSWORD`)
2. `~/.moodle-mcp.json` config file
3. Wizard (interactive, TTY only)

### Wizard flow

```
Welcome to moodle-mcp!

Let's connect to your Moodle in 2 steps.

Step 1 — Your Moodle URL
  Enter the URL your school uses for Moodle
  (e.g. https://moodle.myschool.edu): _

Step 2 — Your API token
  To get your token:
  1. Log into Moodle in your browser
  2. Go to: Profile → Preferences → Security keys
     (opening in browser now...)
  3. Copy the "Moodle mobile web service" token
  Enter token: _

✓ Config saved to ~/.moodle-mcp.json
✓ Fetching your courses...
✓ Obsidian vault created at ~/moodle-vault/

Add this to your Claude Desktop config and restart Claude:

  "moodle": {
    "command": "npx",
    "args": ["moodle-mcp"]
  }

You're all set!
```

- Opens the Moodle security keys page in the default browser automatically using Node's `open` or `child_process` (`start` on Windows, `xdg-open` on Linux, `open` on Mac)
- Saves config to `~/.moodle-mcp.json` as `{ "baseUrl": "...", "token": "..." }`
- Immediately fetches courses and generates the initial Obsidian vault (see section 2)
- Prints the exact Claude Desktop JSON snippet to copy-paste
- Exits — does NOT start the MCP server (user needs to add it to Claude Desktop first)

### Config file format

```json
{
  "baseUrl": "https://moodle.yourschool.edu",
  "token": "abc123"
}
```

Stored at `~/.moodle-mcp.json`. `getConfig()` in `src/config.ts` reads this as a fallback when env vars are absent.

## 2. Obsidian Vault

### Generated by wizard (initial)

After saving config, the wizard fetches courses and writes a vault to `~/moodle-vault/`:

```
~/moodle-vault/
├── .obsidian/
│   └── graph.json          ← pre-configured graph view (colors, filters)
├── Courses/
│   ├── 📚 <Course Name>.md ← links to assignments, files, quizzes
│   └── ...
├── Assignments/
│   ├── <Assignment Name>.md ← links back to course + deadline
│   └── ...
├── Dashboard.md            ← hub note linking all courses (graph center node)
└── _index.md
```

Each note uses `[[wikilinks]]` so Obsidian's graph view shows a real web of connections. Courses link to their assignments; assignments link back to their course and include due dates.

`Dashboard.md` links to every course — it becomes the central node in the graph view.

### `moodle_build_obsidian_vault` MCP tool

Registered in `src/tools/` alongside existing tools. Triggered by Claude on demand (e.g. "refresh my vault", "add this week's notes to Obsidian").

**Input schema:**
```ts
{
  outputPath?: string  // default: ~/moodle-vault
  overwrite?: boolean  // default: true
}
```

**Behavior:** Fetches all courses, assignments, files, and quizzes via existing MCP client methods. Writes/overwrites the vault at `outputPath`. Returns a summary of files written.

**Vault generation logic lives in `src/vault.ts`** — shared between the wizard and the MCP tool so there's no duplication.
